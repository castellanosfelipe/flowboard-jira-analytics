Carga diaria de recursos

Heatmap y tabla ejecutiva de utilización diaria por persona.

Se calcula tomando cada ticket activo con responsable, fecha inicio, fecha fin y estimación original. Primero cuenta los días laborales entre inicio y fin, excluyendo sábados y domingos. Luego distribuye la estimación original en horas entre esos días: carga diaria del ticket = estimación original / días laborales. Finalmente suma la carga diaria de todos los tickets concurrentes por persona y compara contra una capacidad estándar de 8h: % carga = horas asignadas / 8h × 100.

Cubos de envejecimiento

Se calcula con tickets abiertos usando: hoy menos fecha de inicio. Los cubos agrupan 0–7 días como normal, 7–30 días como atención y más de 30 días como riesgo.

Settings

Configuración de conexión con Jira y preferencias.

Estás viendo la versión publicada (GitHub Pages). La conexión en vivo con Jira necesita el proxy local porque Jira bloquea las llamadas directas del navegador (CORS). Para datos reales, clona el repo y ejecuta node server.js. Mientras tanto, prueba el Modo Demo más abajo — funciona aquí sin configurar nada.

URL, usuario y JQL se guardan en este navegador. El token solo se recuerda durante la sesión local actual.

Advertencia de seguridad: este prototipo envía credenciales desde el navegador hacia un proxy local. En producción, el token debe vivir solo en un backend seguro o secret manager.

Sincronización

El proxy local conecta directamente con la API de Jira en cada consulta. No hay caché persistente entre sesiones. Usa Consultar Jira para cargar datos o Refrescar datos para actualizar sin cambiar filtros.

Solo lectura — FlowBoard nunca modifica datos en Jira.
